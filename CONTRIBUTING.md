# Commit guidelines

## Формат сообщения коммита

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type
- `feat`: Новая функция
- `fix`: Исправление ошибки
- `docs`: Изменения в документации
- `style`: Изменения в форматировании кода
- `refactor`: Переструктурирование кода без изменения функциональности
- `perf`: Улучшение производительности
- `test`: Добавление или обновление тестов
- `chore`: Изменения в сборке, зависимостях, конфигурации

### Scope
- `auth` - аутентификация
- `menu` - управление меню
- `orders` - управление заказами
- `payments` - платежи
- `reviews` - отзывы
- `admin` - административные функции
- `ui` - интерфейс
- `db` - база данных

### Примеры

```
feat(auth): добавлена регистрация пользователей

- Реализована форма регистрации
- Добавлена валидация email
- Хеширование пароля с bcryptjs

fix(orders): исправлена ошибка при создании заказа

docs(readme): обновлено описание проекта

style(frontend): форматирование CSS файлов

refactor(backend): оптимизация подключения к БД

perf(menu): добавлено кеширование меню

test(auth): добавлены тесты для авторизации

chore(deps): обновление зависимостей
```

## Git Workflow

### Создание новой ветки
```bash
git checkout -b <type>/<description>
git checkout -b feat/user-registration
git checkout -b fix/order-creation-bug
```

### Коммит изменений
```bash
git add .
git commit -m "feat(auth): добавлена регистрация пользователей"
```

### Push в репозиторий
```bash
git push origin <branch-name>
```

### Pull Request
1. Создайте PR на GitHub
2. Добавьте описание изменений
3. Запросите review
4. После одобрения выполните merge

### Merge в main
```bash
git checkout main
git pull origin main
git merge --no-ff <branch-name>
git push origin main
```

## Ветки репозитория

- `main` - готовый к продакшену код
- `develop` - разработка и интеграция функций
- `feature/*` - новые функции
- `fix/*` - исправления ошибок
- `hotfix/*` - срочные исправления для продакшена

## Правила

1. **Один коммит = одна функция**
   - Не смешивайте логику разных функций в одном коммите

2. **Информативные сообщения**
   - Пишите понятные сообщения коммитов
   - Используйте русский язык

3. **Регулярные коммиты**
   - Коммитьте часто для лучшей истории

4. **Перед push**
   - Убедитесь, что код проходит тесты
   - Проверьте на линтинг ошибки

5. **Актуальная ветка**
   - Регулярно синхронизируйте с main/develop

```bash
git fetch origin
git rebase origin/main
```
